Middleware and Handling in Express.js

Middleware is a key concept in Express.js that allows you to process requests before they reach the route handler, enabling functionality like logging, authentication, and more...

Key concepts for middleware and request handling include:

  • Built-in Middleware: Express provides built-in middleware for parsing requests, serving static files, and more.
  • Custom Middleware: You can create custom middleware functions to add specific functionality to your routes.
  • Request and Response Objects: Middleware functions have access to the request and response objects.

Code Example: Using Middleware in Express.js

Here's an example of using middleware to log incoming requests:

// Import the required modules
        const express = require('express');
        const app = express();
        const PORT = 3000;
        
        // Custom middleware function
        const logRequest = (req, res, next) => {
            console.log(`[${new Date().toISOString()}] ${req.method} ${req.url}`);
            next(); // Call the next middleware or route handler
        };
        
        // Use the custom middleware for all routes
        app.use(logRequest);
        
        // Define a route handler
        app.get('/', (req, res) => {
            res.send('Hello, Express!');
        });
        
        // Start the server
        app.listen(PORT, () => {
            console.log(`Server is running on port ${PORT}`);
        });

This code demonstrates using a custom middleware function `logRequest` to log incoming requests before they reach the route handler.

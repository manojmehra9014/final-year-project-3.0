Working with Streams in Node.js

Streams are a core feature in Node.js that allow you to efficiently read or write data in chunks, making them ideal for handling large files or network data...

Key concepts for working with streams include:

  • Readable Streams: Reading data from a source in chunks using events like 'data' and 'end'.
  • Writable Streams: Writing data to a destination in chunks using events like 'drain' and 'finish'.
  • Piping: Connecting a readable stream to a writable stream to efficiently transfer data.
  • Transform Streams: Combining readable and writable streams to modify data during transfer.

Code Example: Reading and Writing Streams

Here's an example of using streams to read and write data:

// Import the fs module
        const fs = require('fs');
        
        // Create a readable stream
        const readableStream = fs.createReadStream('input.txt', 'utf8');
        
        // Create a writable stream
        const writableStream = fs.createWriteStream('output.txt', 'utf8');
        
        // Pipe the readable stream to the writable stream
        readableStream.pipe(writableStream);
        
        // Handle the 'finish' event
        writableStream.on('finish', () => {
            console.log('Data written successfully.');
        });

This code demonstrates reading data from a file using a readable stream and writing it to another file using a writable stream. The 'pipe' method efficiently transfers data between the two streams.

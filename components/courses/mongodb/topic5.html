Aggregation Framework in MongoDB

The Aggregation Framework in MongoDB provides a powerful set of tools for data processing and analysis. It allows you to perform complex operations on your data, such as grouping, filtering, sorting, and transforming, in a single query.

Aggregation Pipeline

The Aggregation Framework works based on the concept of an aggregation pipeline. The pipeline consists of multiple stages, each performing a specific operation on the data. Some of the commonly used stages include:

  • $match: Filters documents based on specified criteria.
  • $group: Groups documents by a specified key and performs aggregation operations on the grouped data.
  • $sort: Sorts the output documents based on a specified field.
  • $project: Shapes the output documents by specifying which fields to include or exclude.
  • $limit: Limits the number of documents in the output.
  • $skip: Skips a specified number of documents in the output.

Example Aggregation Query

Here's an example of an aggregation query that calculates the average age of documents in a collection:


  db.collection('myCollection').aggregate([
    { $group: { _id: null, averageAge: { $avg: "$age" } } }
  ]);

This query uses the $group stage to group all documents together and calculate the average age using the $avg aggregation operator.
